## Spacing Scale

Declared values (multiples of 4 only). Applied via nativewind utility classes on mobile and Tailwind on web.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, chip inner spacing |
| sm | 8px | Between list items, inline label spacing, horizontal padding inside renewal chips |
| md | 16px | Default screen horizontal padding, card inner padding |
| lg | 24px | Section vertical spacing, form field gaps |
| xl | 32px | Dashboard total card padding, major section breaks |
| 2xl | 48px | Between dashboard sections (total card → renewal strip → list) |
| 3xl | 64px | Bottom safe area padding, tab bar clearance |

Exceptions:
- Touch targets: minimum 44x44px on all interactive elements (ui-ux-pro-max CRITICAL rule). This may exceed the base grid — use 44px height as a floor even when token value is smaller.
- Bottom sheet drag handle: 4px height × 32px width, centered, 8px vertical margin.

## Typography

All sizes in logical pixels (dp on mobile, px on web). System font fallback always applies.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 32px | 600 (semibold) | 1.2 | Monthly total amount ("$247.00"), dashboard hero number |
| Heading | 20px | 600 (semibold) | 1.25 | Screen titles, section headers ("Upcoming Renewals") |
| Body | 16px | 400 (regular) | 1.5 | Subscription names in list, form labels, description text |
| Label | 13px | 400 (regular) | 1.4 | Chips (renewal dates), usage counter ("3 of 5 subscriptions used"), category badges, secondary metadata |

**Weight scale:** Regular (400) and Semibold (600). Two weights only — executor must not introduce additional weights.

**Size scale:** 13, 16, 20, 32. Executor must not introduce additional sizes outside this scale.

**Notes:**
- Display and Heading are both Semibold (600); visual distinction comes from size (32px vs 20px), not weight.
- Minimum body size on mobile is 16px (ui-ux-pro-max CRITICAL rule — readable-font-size).
- Annual toggle label uses Body (16px / 400) — it is a control label, not a heading.
- "Annual" badge next to the toggle uses Label (13px / 400).

## Color

Two complete palettes are required: dark mode (default) and light mode. The app auto-follows the system setting (Claude's discretion — CONTEXT.md). Both palettes share the same structural roles.

### Dark Mode (Default)

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | #0F172A | Main screen background, list backgrounds |
| Secondary (30%) | #1E293B | Cards, subscription list items, bottom sheet background, nav bar |
| Accent (10%) | #22C55E | See accent reserved-for list below |
| Destructive | #EF4444 | Delete subscription action, error states only |
| Text primary | #F8FAFC | All primary text on dark surfaces |
| Text secondary | #94A3B8 | Secondary metadata, placeholder text, muted labels |
| Surface elevated | #334155 | Selected state, active chip, input focus ring |

### Light Mode

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | #F8FAFC | Main screen background |
| Secondary (30%) | #FFFFFF | Cards, subscription list items, bottom sheet background |
| Accent (10%) | #16A34A | See accent reserved-for list below (darkened for contrast on light) |
| Destructive | #DC2626 | Delete subscription action, error states only |
| Text primary | #0F172A | All primary text on light surfaces |
| Text secondary | #475569 | Secondary metadata, placeholder text, muted labels (minimum for 4.5:1 contrast) |
| Surface elevated | #E2E8F0 | Selected state, active chip, input focus state |

**Accent reserved for:**
1. Primary CTA button ("Add Subscription" button fill)
2. Positive financial indicators (monthly total when under a self-set budget — Phase 4, but reserve the semantic now)
3. "Upgrade" paywall CTA button fill
4. Checkmark icon on confirmed subscription items
5. Usage counter bar fill ("3 of 5" progress indicator fill)
6. Annual toggle active state indicator

Accent is NOT used for: navigation tabs, headings, card borders, general text, icons (icons use text-secondary), or the renewal strip chips (chips use surface-elevated with text-primary).

**Annual toggle:** When toggled to "Annual", the display amount updates. The toggle control uses accent for the active/on state thumb.

## Screens and Interaction States Required — Phase 1

This section maps every screen, state, and interaction the executor must implement.

### Auth Screens

| Screen | States Required |
|--------|----------------|
| Sign Up | Default, loading (button disabled with spinner), email already exists error |
| Email Verification | Pending state (instructions), resend link CTA |
| Log In | Default, loading, wrong credentials error |
| Session Restore | Transparent splash with app icon (no loading spinner on dashboard — skeleton instead) |

### Dashboard Screen

| Element | States Required |
|---------|----------------|
| Total card | Loaded (monthly), loaded (annual — toggle active), loading skeleton |
| Subscription list | Loaded (sorted by cost desc), empty state, loading skeleton |
| Renewal strip | Loaded (5-7 chips), empty (no upcoming — strip hidden, not empty state), loading skeleton |
| Usage counter | Free tier: "N of 5 subscriptions used" + progress bar; paid tier: counter hidden |
| Annual toggle | Off (monthly, default), on (annual) — toggle uses accent color |

**Dashboard sort order:** Highest cost first (D-02). Within same cost, alphabetical by service name.

**Loading skeleton:** Match the shape of each element it replaces. Use animated shimmer (opacity pulse 1.0 → 0.4 → 1.0 at 1.2s). Do not show a global spinner — skeleton per section.

### Empty State (Dashboard — no subscriptions)

Displayed when the authenticated user has zero subscriptions.

| Element | Spec |
|---------|------|
| Illustration | Single-color line art, service-icon cluster motif (Netflix/Spotify/Apple-style logos in a soft arrangement). Style: clean, minimal, not cartoon. Color: text-secondary opacity 0.4 on dominant background. Size: 160×160dp. |
| Heading | "No subscriptions yet" |
| Body | "Add your first to start tracking what you spend." |
| CTA button | "Add Subscription" — accent fill, Body weight semibold, full-width, 48px height |
| Placement | Vertically centered in the list area, below the total card and renewal strip (strip hidden when empty) |

### Add Subscription Form (Modal / Full-Screen Sheet)

| Field | Type | Required | Spec |
|-------|------|----------|------|
| Service name | Text input with auto-suggest | Yes | Typeahead from logo library (top 150-200 services); shows service logo thumbnail in suggestion row; selects auto-fill amount + category + frequency if known |
| Amount | Numeric input (currency) | Yes | Prefix "$" symbol; numeric keyboard; 2 decimal places |
| Billing frequency | Segmented control | No | Monthly / Yearly / Weekly / Custom |
| Next renewal date | Date picker | No | Defaults to 30 days from today if frequency = Monthly |
| Category | Icon picker grid | No | 6 categories: Entertainment, Productivity, Health, Finance, Shopping, Other; each has a distinct icon from @expo/vector-icons |

**Auto-suggest behavior:** Suggestions appear after 2 characters typed. Maximum 5 suggestions shown. Each row: service logo (24×24dp) + service name (Body) + known amount (Label, text-secondary). Tap to select fills all known fields instantly.

**Required field validation:** Name and amount fields show inline error beneath the field (text-destructive, Label size) if user attempts to save with empty value. Other fields never show validation errors.

**Save button label:** "Add Subscription" — accent fill, full-width, 48px height, Body semibold. Disabled (opacity 0.4) until name + amount are both filled.

### Edit Subscription Screen

Reuses Add form layout. Pre-populates all existing values.

| Element | Spec |
|---------|------|
| Primary action | "Save Changes" — accent fill |
| Delete/Archive action | "Delete Subscription" — text-only, text-destructive color, positioned below Save with 24px gap |

### Delete Confirmation

Triggered from Edit screen "Delete Subscription" tap.

| Element | Spec |
|---------|------|
| Pattern | Bottom action sheet (not a modal dialog) |
| Title | "Delete [Service Name]?" |
| Body | "This will permanently remove it from your tracker." |
| Confirm action | "Delete Subscription" — destructive fill (#EF4444 / #DC2626), full-width, 48px |
| Cancel action | "Cancel" — secondary surface fill, full-width, 48px, above Confirm |
| Animation | Slides up (react-native-reanimated), backdrop dims to 60% opacity |

### Paywall Bottom Sheet (D-06)

Triggered when free user attempts to add a 6th subscription.

| Element | Spec |
|---------|------|
| Trigger | "Add Subscription" form open attempt when count = 5 |
| Pattern | Bottom sheet, not full-screen modal |
| Title | "You've reached your limit" |
| Body | "Free accounts track up to 5 subscriptions. Upgrade to track unlimited." |
| Feature list | 3 bullets: "Unlimited subscriptions", "Auto-detection via Gmail (coming soon)", "Bank sync (coming soon)" |
| Primary CTA | "Upgrade to Pro" — accent fill, full-width, 48px |
| Secondary CTA | "Maybe later" — text-only, text-secondary, 44px touch target |
| Animation | Slides up from bottom (react-native-reanimated spring: damping 20, stiffness 150). Backdrop: 60% opacity. Drag-to-dismiss enabled. |

### Notification Preferences Screen (Settings)

| Element | Spec |
|---------|------|
| Toggle: renewal reminders | Label: "Renewal Reminders" / Description: "Get notified 3 days before a subscription renews" / Toggle: system-style (accent on-state) |
| State saved | Immediately on toggle change (no Save button) |

## Animation and Transition Contracts

| Interaction | Duration | Easing | Library |
|-------------|----------|--------|---------|
| Bottom sheet open (paywall, delete) | 320ms | Spring: damping 20, stiffness 150 | react-native-reanimated |
| Bottom sheet close | 250ms | Spring: damping 25, stiffness 200 | react-native-reanimated |
| Annual toggle flip (amount change) | 180ms | ease-in-out | react-native-reanimated |
| Screen transition (Expo Router) | 300ms | Default stack slide | expo-router default |
| Loading skeleton shimmer | 1200ms loop | linear | react-native-reanimated opacity |
| Auto-suggest list appear | 150ms | ease-out | react-native-reanimated height expand |
| Button press feedback | scale 0.97, 100ms | ease-in-out | react-native-reanimated |
| List item swipe (delete reveal) | gesture-driven | spring | react-native-reanimated + gesture-handler |

**prefers-reduced-motion:** Check `AccessibilityInfo.isReduceMotionEnabled()` on mount. When true: disable shimmer loop (static skeleton), remove spring animations (instant), remove scale press feedback. Duration-based transitions may remain at 50ms.

## Accessibility Requirements (Phase 1 Baseline)

Per ui-ux-pro-max CRITICAL rules:

- All touch targets minimum 44×44dp (including chips, toggle, icon-only buttons)
- Text contrast: minimum 4.5:1 for normal text in both light and dark mode
- Every icon-only button must have `accessibilityLabel` prop
- Form inputs must have associated labels (not just placeholder text)
- FlatList items must have `accessibilityRole="button"` if tappable
- `AccessibilityInfo.isReduceMotionEnabled()` must gate all animations
